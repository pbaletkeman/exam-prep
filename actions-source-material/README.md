## Format

Every file follows a consistent structure:

- **Metadata block** — iteration number, generation date, total questions, difficulty split, and answer type distribution
- **Questions section** — 100 questions, each with:
  - A category header (`### Question N — Category`)
  - `**Difficulty**`, `**Answer Type**`, and `**Topic**` fields
  - An optional `**Scenario**` block for context-based questions
  - Four answer choices (A–D), or more for `many`/`all` type questions
- **Answer Key** — a table with columns `Q# | Answer(s) | Explanation | Source | Difficulty`, referencing one of the 19 source topic files

### Answer Types

| Type | Meaning |
|------|---------|
| `one` | Exactly one correct answer |
| `many` | Two or more correct answers (select all that apply) |
| `all` | All listed options are correct |
| `none` | No listed option is correct |

## Suggested Study Approach

1. **Start with iteration 1** — it is the reference format and covers all topics evenly.
2. **Use iterations 2–6** for mixed answer type practice, which mirrors real exam variability.
3. **Use iterations 7, 9, 10** for Medium-weighted single-select drilling.
4. **Use iteration 8** as a final challenge exam — it has the highest Hard question ratio.
5. Cross-reference the **Source** column in each answer key to go deeper on topics you miss.
